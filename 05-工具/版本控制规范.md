# 版本控制规范

> Git版本控制标准操作流程

---

## 分支策略

```
main                    # 主分支，稳定版本
├── develop             # 开发分支
├── feature/vol1-ch01   # 特性分支：第一卷第1章
├── feature/vol1-ch02   # 特性分支：第一卷第2章
├── hotfix/typo-fix     # 热修复分支
└── backup/20260321     # 备份分支
```

## 提交规范

### 提交信息格式
```
<类型>: <简短描述>

<详细描述>
```

### 类型说明
- `feat`: 新章节/新功能
- `fix`: 修复错误
- `docs`: 文档更新
- `style`: 格式调整
- `refactor`: 重构
- `chore`: 构建/工具

### 示例
```
feat: 完成第一卷第1章初稿

- 字数: 7500字
- 完成3秒伏笔植入
- 待审核
```

## 工作流程

1. **创建特性分支**
   ```bash
   git checkout -b feature/vol1-ch01
   ```

2. **开发和提交**
   ```bash
   git add .
   git commit -m "feat: 完成第1章初稿"
   ```

3. **合并到develop**
   ```bash
   git checkout develop
   git merge feature/vol1-ch01
   ```

4. **合并到main**
   ```bash
   git checkout main
   git merge develop
   git push origin main
   ```
