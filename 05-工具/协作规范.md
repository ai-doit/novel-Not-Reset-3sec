# 协作规范

> 多人协作时的工作流程和沟通规范

---

## 角色分工

| 角色 | 职责 |
|------|------|
| 主作者 | 最终决策、质量把控 |
| Agent操作员 | 启动Agent、监控进度 |
| 审核员 | 人工审核关键章节 |
| 技术维护 | 环境维护、故障处理 |

## 沟通渠道

- **日常同步**: Feishu群
- **进度汇报**: 每日17:00自动报告
- **紧急问题**: 直接电话/语音
- **文档协作**: GitHub + Obsidian

## 冲突解决

1. **文件冲突**: 使用Git合并工具
2. **创意分歧**: 主作者最终决策
3. **技术问题**: 参考故障排除手册

## 代码审查

- 关键章节必须人工审核
- 使用Pull Request流程
- 至少1人审核通过才能合并
