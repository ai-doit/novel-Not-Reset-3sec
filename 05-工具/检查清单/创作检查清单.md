# 创作检查清单

> 用于检查章节质量、一致性和完成度的标准化清单

---

## 📝 章节创作检查清单

### 基础检查（必须全部通过）

- [ ] **字数达标**: 达到目标字数的 ±10% 范围内
- [ ] **章节编号正确**: 与大纲中的章节号一致
- [ ] **章节标题**: 包含编号和名称（如：第001章-七千米）
- [ ] **视角一致**: 整章保持同一人物视角，无突然切换
- [ ] **时态一致**: 保持过去时或现在时，不混用

### 内容检查（必须全部通过）

- [ ] **核心事件完整**: 大纲中的所有核心事件都已包含
- [ ] **3秒伏笔已植入**: 至少一处"3秒"主题的伏笔或呼应
- [ ] **情感落点到位**: 章节结尾有明确的情感高潮或悬念
- [ ] **场景描写充分**: 关键场景有足够的感官细节
- [ ] **对话自然**: 人物对话符合其性格特征，不过度解释

### 角色检查（必须全部通过）

- [ ] **林野言行一致**: 冷静、摸婚戒、做决定前数秒
- [ ] **小陈言行一致**: 乐观、紧张时数秒、口头禅"我来"
- [ ] **苏晚言行一致**: 愧疚、看女儿照片、提及女儿情绪波动
- [ ] **新角色已记录**: 如有新角色，已添加到角色数据库
- [ ] **人物关系清晰**: 人物之间的互动符合其关系设定

### 技术检查（必须全部通过）

- [ ] **无错别字**: 已完成拼写检查
- [ ] **标点正确**: 标点符号使用规范
- [ ] **段落长度适中**: 没有超过5行的长段落
- [ ] **格式一致**: Markdown格式规范，标题层级正确
- [ ] **链接有效**: 如有内部链接（[[...]]），目标存在

## 🔍 卷级审核检查清单

### 剧情连贯性（每卷完成后检查）

- [ ] **卷首衔接**: 与上一卷的结尾自然衔接
- [ ] **卷终悬念**: 为下一卷留下合适的悬念或铺垫
- [ ] **节奏把控**: 高潮和低谷分布合理，无拖沓或跳跃
- [ ] **支线整合**: 普通人视角支线与主线融合自然
- [ ] **情感戏到位**: 至少1章纯情感戏，情感落点充分

### 人物弧光追踪（每卷完成后检查）

- [ ] **林野弧光推进**: 从X阶段推进到Y阶段，符合大纲
- [ ] **小陈弧光推进**: 从X阶段推进到Y阶段，符合大纲
- [ ] **苏晚弧光推进**: 从X阶段推进到Y阶段，符合大纲
- [ ] **普罗米修斯弧光**: 五阶段推进符合设定
- [ ] **行为一致性**: 所有角色的行为与其弧光阶段匹配

### 伏笔线索审核（每卷完成后检查）

- [ ] **3秒伏笔递进**: 从0.3秒 → 3秒 的递进关系清晰
- [ ] **三源线索埋置**: 碎片/三源树/音频的线索已埋置
- [ ] **三层真相铺垫**: 表层/中层/深层真相的铺垫到位
- [ ] **前卷伏笔回收**: 前一卷埋置的伏笔在本卷回收
- [ ] **本卷伏笔埋置**: 为后续卷埋置新的伏笔

### 设定一致性（每卷完成后检查）

- [ ] **碎片设定一致**: 碎片的功能、特性、出现时机统一
- [ ] **AI能力一致**: 普罗米修斯的能力、限制、进化阶段统一
- [ ] **科技设定自洽**: 所有科技元素设定前后一致
- [ ] **世界观统一**: 文明循环、熵、灵基等设定统一
- [ ] **时间线正确**: 事件顺序、时间跨度、人物年龄正确

### 全本整合前检查

- [ ] **六卷全部完成**: 所有六卷和终章都已定稿
- [ ] **总字数达标**: 100万字 ± 5%

- [ ] **对话自然流畅**: 所有对话经过润色，自然流畅
- [ ] **场景描写生动**: 关键场景描写生动，有画面感
- [ ] **情感戏到位**: 所有情感戏情感真挚，打动人心
- [ ] **节奏张弛有度**: 全本节奏把控得当，无拖沓或仓促

### 技术细节检查

- [ ] **格式一致**: Markdown格式统一，标题层级正确
- [ ] **链接有效**: 所有内部链接有效，无死链
- [ ] **图片正常**: 如有图片，全部正常显示

### 出版准备检查

- [ ] **目录完整**: 自动生成或手动制作的目录完整
- [ ] **页眉页脚**: 页眉页脚设置正确
- [ ] **页码连续**: 页码连续无断号
- [ ] **章节分页**: 每章从新页开始
- [ ] **字体嵌入**: PDF中文字体已嵌入

## 📝 使用说明

### 日常创作使用

1. **章节完成后**: 对照"章节创作检查清单"自检
2. **卷完成后**: 对照"卷级审核检查清单"审核
3. **全本完成后**: 对照"最终交付检查清单"检查

### 检查方式

- **自检**: 作者自己检查，打勾确认
- **交叉检查**: 其他Agent或人工检查
- **最终审核**: 主协调Agent确认

### 检查记录

每次检查完成后，在 `00-项目管理/检查记录/` 中创建记录文件：

```markdown
# 检查记录 - 第X卷

- **检查日期**: 202X-XX-XX
- **检查类型**: 卷级审核
- **检查人**: [Agent名称或人工]
- **检查结果**: [通过/需修改]

## 发现的问题

- [ ] 问题1: [描述] → [处理方式]
- [ ] 问题2: [描述] → [处理方式]

## 已确认通过的项目

- [x] 项目1
- [x] 项目2
```
